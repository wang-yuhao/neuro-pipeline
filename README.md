# 🧠 Autonomous Computational Neuroscience Research Pipeline

A fully automated, 6-agent AI research pipeline that takes a JSON configuration file
and produces a complete research codebase, experimental results, and a publication-ready
manuscript draft — with minimal human intervention.

---

## Architecture Overview

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                          ORCHESTRATOR (orchestrator.py)                       │
│  Manages flow, gate decisions, revision cycles, audit logging, and failover   │
└────────────────────────────────┬─────────────────────────────────────────────┘
                                 │
         ┌───────────────────────▼──────────────────────────┐
         │            REVISION LOOP (max 2 cycles)           │
         │  ┌──────────────────┐    ┌──────────────────────┐ │
         │  │  Agent 1         │◄───│  Agent 2             │ │
         │  │  Topic Proposer  │    │  Peer Reviewer       │ │
         │  │  proposal.json   │    │  (score ≥7.0 → pass) │ │
         │  └──────────────────┘    └──────────────────────┘ │
         └───────────────────────┬──────────────────────────┘
                                 │ APPROVED
         ┌───────────────────────▼──────────────────────────┐
         │          VALIDATION LOOP (max 2 cycles)           │
         │  ┌──────────────────┐    ┌──────────────────────┐ │
         │  │  Agent 3         │◄───│  Agent 4             │ │
         │  │  Experiment      │    │  Logic & Resource    │ │
         │  │  Designer        │    │  Validator           │ │
         │  └──────────────────┘    └──────────────────────┘ │
         └───────────────────────┬──────────────────────────┘
                                 │ VALIDATED
         ┌───────────────────────▼──────────────────────────┐
         │            DEBUG LOOP (max 5 cycles)              │
         │  ┌──────────────────────────────────────────────┐ │
         │  │  Agent 5 — Developer & Researcher            │ │
         │  │  Generates full Python codebase              │ │
         │  │  Runs smoke test + unit tests                │ │
         │  │  Auto-debugs on failure                      │ │
         │  └──────────────────────────────────────────────┘ │
         └───────────────────────┬──────────────────────────┘
                                 │ TESTED
         ┌───────────────────────▼──────────────────────────┐
         │  Agent 6 — Academic Writing Agent               │
         │  Produces publication-ready manuscript draft     │
         └──────────────────────────────────────────────────┘
```

## Deliverables

| File | Description | Generated By |
|------|-------------|--------------|
| `outputs/proposal.json` | Research proposal with hypothesis, background, RQs | Agent 1 |
| `outputs/feasibility_review.json` | Peer review scores and decision | Agent 2 |
| `outputs/experiment_design.json` | Complete experiment specification | Agents 3+4 |
| `outputs/codebase/` | Full Python research repository | Agent 5 |
| `outputs/results.json` | Experiment metrics and statistics | Agent 5 |
| `outputs/test_report.json` | Smoke test + unit test results | Agent 5 |
| `outputs/manuscript_draft.md` | Publication-ready manuscript | Agent 6 |
| `outputs/pipeline_log.json` | Full audit trail with timestamps | Orchestrator |

## Prerequisites

- **Python 3.10+**
- **Anthropic API key** with access to Claude Sonnet 4 (claude-sonnet-4-20250514)
- **GPU** (optional but recommended for smoke test) — NVIDIA CUDA or Apple Silicon
- **Disk space**: ~2GB for outputs
- **Internet access**: for API calls and dataset downloads

## Setup

### Step 1: Clone / Extract the project
```bash
cd neuro_pipeline/
```

### Step 2: Set your API key
```bash
export ANTHROPIC_API_KEY="your-key-here"
# OR create a .env file:
echo "ANTHROPIC_API_KEY=your-key-here" > .env
```

### Step 3: Run setup
```bash
bash setup.sh
```
This creates a virtual environment and installs all dependencies.

### Step 4: (Optional) Customize research topics
Edit `config/init_params.json` to specify:
- Your candidate research topics
- Available computational resources
- Target journals
- Reference papers

---

## Running the Pipeline

```bash
# Activate environment
source venv/bin/activate

# Run with default config
python orchestrator.py

# Run with custom config
python orchestrator.py --config config/my_custom_params.json
```

## Orchestration Rules

1. **Gate decisions**: Each agent must pass its quality gate before handoff
2. **Revision cycles**: Agent 1↔2 and Agent 3↔4 loops have configurable max cycles
3. **Debug cycles**: Agent 5 auto-debugs up to 5 times if smoke test fails
4. **Failure escalation**: After 3 consecutive failures, writes `human_escalation_report.txt` and pauses
5. **Audit trail**: Every decision logged to `pipeline_log.json` with timestamp and output hash
6. **Schema validation**: Every agent validates upstream output before processing

## Extending the Pipeline

### Adding a new agent
1. Create `agents/agent7_yourname.py` following the existing pattern
2. Add the agent call in `orchestrator.py`
3. Define the output schema in `utils/schema_validator.py`

### Changing the model
Edit `utils/api_client.py` → `MODEL` constant.

### Adjusting review thresholds
Edit `config/init_params.json` → `pipeline_settings.approval_threshold`

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `ANTHROPIC_API_KEY not set` | Export the env var or create `.env` file |
| `Rate limit errors` | The API client retries automatically; if persistent, check your tier |
| `Smoke test fails repeatedly` | Check `outputs/test_report.json` for errors; Agent 5 auto-debugs |
| `Agent 2 rejects proposal` | Increase `max_revision_cycles` or provide better topics in init_params |
| `JSON parse errors` | Rare model output issue; the orchestrator will retry automatically |
| `pip install fails` | Ensure Python 3.10+ and internet access; check `requirements.txt` |
